1. Vue d'ensemble

1.1 Objectif du module

Le module "Test d'Éligibilité VISA" permet aux étudiants internationaux d'évaluer leurs chances d'obtenir un visa étudiant pour différents pays (France, Canada, USA, UK, etc.).

Caractéristiques principales :
  • Scoring mathématique pour le calcul du score
  • IA (Perplexity/Gemini) pour la génération de briefings personnalisés
  • Scraper automatique pour récupérer les exigences officielles des pays
  • Chatbot d'assistance avec limitation

1.2 Format Backoffice MMS

⚠️ IMPORTANT : Les questions doivent utiliser le Builder MMS avec la structure :
- language + title/title_en
- code (unique) + score (poids)
- options avec title/title_en/code/score
Voir section 3.2 pour le format complet du Builder MMS.

1.3 Différences avec le test MMS existant

Caractéristique Test MMS actuel Module VISA
Scoring 100% mathématique Mixte (math + IA pour briefing)
Questions À choix fixes Choix + entrées texte + numériques
Budget Valeur fixe Comparaison avec seuils pays
Sources de données Statiques Scraper dynamique
Chatbot Non Oui (avec limitation)
Base de données MMS existante MMS existante (même DB, collections étendues)

3. Structure des questions

Le test comprend 23 questions organisées en 6 catégories :

Pays couverts :
  • Q1 (Nationalité) : 47 pays dont Maroc, Tunisie, Algérie, Égypte, Arabie saoudite, Émirats, Nigeria, Afrique du Sud, Inde, Chine, Corée du Sud, Japon, etc.
  • Q2 (Destination) : 19 pays d'étude : USA, Canada, Australie, UK, France, Allemagne, Espagne, Italie, Finlande, Irlande, Danemark, Norvège, Suède, Pologne, Malte, Belgique, Chine, Turquie, Corée du Sud
Catégorie Questions Poids total
Personnel Q4-Q6 25 pts
Éducation Q8, Q10-Q12 20 pts
Langue Q9 8 pts
Projet Q13-Q15 14 pts
Finance Q16-Q20 33 pts
Total 20 questions scorées 100 pts

3.1 Types de questions

  • single_choice : Une réponse parmi plusieurs options
  • multi_choice : Plusieurs réponses possibles
  • boolean : Oui/Non
  • number : Entrée numérique
  • text : Texte libre (non scoré, informationnel)

3.2 Format MMS Builder

Interface du Backoffice MMS - Eligibility Tests :
  • Page : Eligibility Tests → Liste des tests
  • Filtres : Par nom/description, Type, Study Levels, Program Domains
  • Actions : Add Test, Edit, Open Builder

3.2.1 Structure d'une question (JSON)

{
  "language": "fr",           // 🇫🇷 Langue de l'interface (pour le builder)
  "title": "Libellé FR",       // Texte de la question en français
  "title_en": "English text",  // Texte de la question en anglais
  "code": "Q_unique_key",      // Code unique de la question
  "score": 10,                // Score affiché dans le builder (référence)
  "description": "Description FR",
  "description_en": "Description EN",
  "category": "personal",      // Catégorie (voir liste ci-dessous)
  "type": "single_choice",     // single_choice, multi_choice, boolean, number, text
  "options": [                // Options de réponse
    {
      "title": "Option FR",
      "title_en": "Option EN",
      "code": "OPT_01",
      "score": 80              // ⚠️ Score 0-100 : % de points accordés
    }
  ]
}
⚠️ IMPORTANT - Système de scoring MMS :
Les scores des options sont sur une échelle 0-100 (pourcentage).
Score final de la question = (Score option / 100) × Points max de la question

Exemple : Question avec 10 points max, option score 80 → (80/100) × 10 = 8 points

Comprendre le système de scoring MMS (0-100) :

│  Question Q4 - Historique visa                              │
│  Score (shared) = 10  ← Points MAX de la question          │
│                                                             │
│  Options (scores sur 100 = pourcentage) :                   │
│  ┌────────────────────────────────────┬─────────────┐      │
│  │ Option                              │ Score MMS   │      │
│  ├────────────────────────────────────┼─────────────┤      │
│  │ Déjà obtenu                         │ 100         │      │
│  │ Jamais                              │ 80          │      │
│  │ Refusé - motif corrigé              │ 60          │      │

│  └────────────────────────────────────┴─────────────┘      │
│                                                             │
│  Calcul : (Score option / 100) × Score question            │
│  Ex: (80/100) × 10 = 8 points                              │

3.3 Questions conditionnelles

Questions avec affichage conditionnel :
  • Q3bis (autorisation parentale) : uniquement si Q3 = "Oui" (mineur)
  • Q3ter (tranche d'âge) : uniquement si Q3 = "Non" (majeur)
  • Q9bis (détails test langue) : uniquement si Q9 = "Oui"

Dans MMS, utiliser les sections/étapes ou les champs conditionnels du Builder.

3.4 Question avec calcul dynamique (Budget Q18)

Le scoring de Q18 compare le montant saisi au minimum requis par le pays : 
ratio = montantUtilisateur / montantRequisPays
if (ratio >= 1) points = 10
else if (ratio >= 0.8) points = 6
else if (ratio >= 0.6) points = 3
else points = 0 + HARD_FAIL

4. Système de scoring

4.1 Formule de calcul

Score brut = Σ(points obtenus par question)
Score max = Σ(poids des questions scorées)
Score normalisé = (Score brut / Score max) × 100

4.2 Règles de scoring par type

Type 1 : Choix avec règles explicites

Exemple Q4 (Historique visa) :

scoringRules: {
  "Jamais": 8,
  "Déjà obtenu": 10,
  "Déjà refusé - motif corrigé": 6,
  "Déjà refusé - non corrigé": 0
}
Points = (scoringRules[réponse] / 10) × weight

Type 2 : Boolean

if (réponse === true) points = weight
else points = 0

Type 3 : Numérique avec comparaison

ratio = montantUtilisateur / montantRequisPays
if (ratio >= 1) points = 10
else if (ratio >= 0.8) points = 6
else if (ratio >= 0.6) points = 3
else points = 0 + hardFail

4.3 Points bloquants (Hard Fails)

Certains critères provoquent un échec automatique :
  • Refus de visa non corrigé → Score max = 40/100
  • Antécédents migratoires graves → Score max = 40/100
  • Casier judiciaire grave → Score max = 40/100
  • Mineur sans autorisation parentale → Score max = 40/100
  • Budget < 60% du requis → Score max = 40/100

4.4 Seuils de statut

Score Statut Signification
≥ 80% ELIGIBLE Profil solide
65-79% MITIGE Profil moyen avec points à améliorer
50-64% FAIBLE Manque des critères importants
< 50% A_RISQUE Risque élevé de refus
